# Hackathon Requirements

## Event Info

- **Event:** Google Cloud Rapid Agent Hackathon
- **Track:** MongoDB
- **Deadline:** June 11, 2026 @ 2:00 PM PDT (June 12 @ 4:00 AM GMT+7)
- **Prize:** $5,000 (1st place in MongoDB track)
- **Submission:** Devpost

---

## What the Hackathon Wants

### Core Requirements

1. **Build a functional agent** — powered by Gemini and Google Cloud Agent Builder (or ADK)
2. **Integrate a partner MCP server** — MongoDB MCP for this project
3. **Solve a real-world challenge** — event-driven business marketing
4. **Move beyond chat** — agent must take actions, not just answer questions
5. **Multi-step mission** — agent handles complex goals, plans steps, uses tools
6. **Partner Power** — meaningful integration with MongoDB via MCP

### Submission Requirements

1. **URL to open-source code repository** — GitHub repo with MIT license
2. **URL to hosted project** — Cloud Run deployment
3. **~3-minute demo video** — YouTube or Vimeo
4. **Selected partner track** — MongoDB
5. **Completed Devpost submission form**

## What Sets Us Apart

### From Other MongoDB Submissions

Most MongoDB submissions will be:
- Generic data viewers
- Simple CRUD apps
- Chatbots that answer questions
- Data analysis tools

EventBoost is different because:
- **Agent takes actions** — not just suggestions
- **Real business value** — measurable ROI
- **Multi-step workflow** — event → promo → prediction → inventory → staff → marketing
- **Self-improvement** — agent learns from each event
- **Tangible output** — QR codes, promos, social media posts

### From Other Hackathon Submissions

- **Not a chatbot** — agent executes, not just answers
- **Not a toy** — solves real business problem
- **Not generic** — event-specific, location-aware
- **Not static** — improves over time

## Common Mistakes to Avoid

1. **Agent just answers questions** — Must take actions
2. **No MCP integration** — Must use MongoDB MCP meaningfully
3. **No demo video** — Required for submission
4. **Private repo** — Must be public
5. **No license** — Must have visible MIT license
6. **Not working** — Must be installable and runnable
7. **Too complex** — Keep demo simple and clear
8. **No impact story** — Show ROI, before/after comparison
